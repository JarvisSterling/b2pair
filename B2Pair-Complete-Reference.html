1. Platform Overview & User Roles

B2Pair is an AI-powered B2B event matchmaking platform enabling organizers to create events, manage participants/sponsors/exhibitors, and provide AI-driven matchmaking, meeting scheduling, and real-time messaging.

RoleDescriptionEntry PointDashboard
OrganizerCreates/manages events, configures AI matching, manages participants & partnersSign up ‚Üí /onboarding/dashboard/w/[workspaceId]
ParticipantAttends events, gets AI matches, books meetings, messages/events/[slug]/register/dashboard/events/[id]
SponsorCompany funding/brand visibility with tiered packagesInvite link ‚Üí /events/[slug]/partners/onboard/[code]/dashboard/company/[id]
ExhibitorCompany with booth for product showcase & lead captureInvite link ‚Üí /events/[slug]/partners/onboard/[code]/dashboard/company/[id]

A company can be both Sponsor and Exhibitor simultaneously.

30. AI & Matching Engine (Technical)

30.1 Intent Engine

6 intents: buying, selling, investing, partnering, learning, networking. Max 3 per participant.

Intent Compatibility Matrix

BuyingSellingInvestingPartneringLearningNetworking
BuyingLowHighLowMediumLowMedium
SellingHighLowMediumMediumLowMedium
InvestingLowMediumLowHighLowMedium
PartneringMediumMediumHighMediumMediumHigh
LearningLowLowLowMediumLowMedium
NetworkingMediumMediumMediumHighMediumMedium

30.2 Embedding Generation

Model: OpenAI text-embedding-3-small (1536 dimensions). Input text constructed from: title, company, bio, intent, looking_for, offering, expertise areas, interests. Stored in profile_embeddings table using pgvector.

30.3 Match Scoring Algorithm

For each participant pair (A, B):

  1. Intent Score ‚Äî Computed from compatibility matrix between A's intents and B's intents
  2. Industry Score ‚Äî 100 if same industry, partial for related industries
  3. Interest Score ‚Äî Jaccard similarity of expertise areas + interests
  4. Complementarity Score ‚Äî Cosine similarity of embedding vectors
  5. Final Score = (intent √ó weight) + (industry √ó weight) + (interest √ó weight) + (complementarity √ó weight), normalized to 0‚Äì100

Pairs below minimum_score are discarded. Max recommendations per participant capped. Sponsor/VIP boost applied if enabled.

35. Event Lifecycle

  1. Organizer signs up ‚Üí Account created ‚Üí Platform onboarding (4 steps)
  2. Creates workspace ‚Üí Organization record created
  3. Creates event ‚Üí Event details, type, format, dates, venue, settings
  4. Configures ‚Üí Matching rules, participant types with permissions
  5. Builds event page ‚Üí Drag-and-drop editor with 13 block types, 3 themes
  6. Adds partners ‚Üí Creates sponsor/exhibitor companies with invite codes
  7. Publishes event ‚Üí Status: published, registration link active
  8. Partners onboard ‚Üí Complete 7-step wizard, submit for review
  9. Organizer reviews ‚Üí Approve/reject partners (with reason)
  10. Participants register ‚Üí 2-step flow via public event page
  11. Organizer approves ‚Üí If requires_approval, manually approve registrations
  12. AI generates embeddings ‚Üí OpenAI text-embedding-3-small vectors created
  13. AI generates matches ‚Üí 4-factor scoring algorithm runs on all pairs
  14. Participants discover ‚Üí View matches, save/dismiss, request meetings
  15. Meetings scheduled ‚Üí Auto-scheduling with mutual availability
  16. Pre-event messaging ‚Üí 1:1 chat with file sharing
  17. Event day ‚Üí QR check-in, meetings, live messaging, contact exchange
  18. Post-event ‚Üí Meeting ratings (1-5 stars + feedback), analytics review
  19. ROI review ‚Üí Sponsors/exhibitors check leads, analytics, team performance
